# 对公客户深挖分析

## 使用方式

用户给公司名称 → 按下述步骤执行 → 输出标准分析报告。

**模型选择**：批量任务统一使用 `minimax-m2.7`，不使用 Codex（除非用户明确指定）。

批量任务时，默认规则如下：
- 对名单中的**每一家公司都要查**，不自行缩减范围
- 分批执行仅用于控频、防风控、提效率，**不代表减少覆盖范围**
- 若服务器检索路径持续失败、结果质量差、频繁触发验证码/风控，切换到 `browser-relay`，通过用户本地浏览器继续逐家公司补齐
- 交付物只保留整理后的分析结果；同步仓库时不混入原始搜索缓存、临时抓取文件等无关产物
- **每个批次必须生成独立 INDEX 文件**，含评级统计、超链接、可点击评级列

## 执行步骤

### 第一步：读取分析模板
读取 `references/analysis-template.md`，获取完整分析框架和评分标准。

### ⚠️ 第二步（必做）：判断企业性质 — 先定类型再分析

**在填任何分析之前，先判断这个企业是哪种类型：**

| 企业性质 | 特征 | 分析逻辑 |
|---------|------|---------|
| **运营实体** | 有实际业务、员工、产品 | 按正常经营逻辑分析 |
| **控股公司/母公司** | 员工极少或为0、持有其他公司股权、注册资本比子公司小 | **不能看自身，要看持有的子公司价值** |
| **混合型** | 部分自营+部分股权投资 | 拆分两部分分别分析 |

**控股公司/母公司的判断标准（满足3条及以上即判定）：**
- [ ] 是否有大量对外投资（持股子公司）？
- [ ] 是否基本无实际业务运营？
- [ ] 员工人数是否为0或极低？
- [ ] 经营范围是否主要是"投资""企业管理""咨询"？
- [ ] 注册资本是否比被投资的子公司还小？

**⚠️ 控股公司/母公司的评分规则：**
- 自身"员工为0""注册资本小" → **不扣分，正常现象**
- 控股公司评分 = 其持有子公司的综合评分（参考子公司行业、融资、成长性）
- 子公司价值高 → 控股公司评级可参考子公司，最高至A类

**⚠️ 分析标的为子公司时的评分规则：**
- 子公司自身评级 + 母公司辐射价值都要看
- 若母公司在名单里或母公司体量大/有融资 → 可作为"以大带小"切入点，子公司评级可上浮

### 第三步：搜索企业基础信息
默认使用 `agent-browser` 打开搜索引擎检索；若服务器环境受限、结果明显不全、连续出现验证码/风控，改用 `browser-relay` 走用户本地 Chromium 继续。

基础查询词可从以下几类组合切换，避免单一重复模式：
```
北京+[公司名称]+公司
[公司名称]+工商
[公司名称]+官网
[公司名称]+爱企查
[公司名称]+天眼查
```
重点抓取：
- 爱企查/天眼查的企业信息卡片（工商信息、注册资本、法人、成立时间）
- 企业性质（民营/国企/上市公司/外资）
- 高新技术企业、专精特新认定
- 主营业务描述
- 企业地址和联系方式

### 第四步：搜索业务和行业信息
搜索：
```
北京+[公司名称]+招投标
北京+[公司名称]+主营+行业
[公司名称]+中标
[公司名称]+产品
```
重点抓取：
- 近期招投标记录（近6-12个月）
- 主营业务和行业赛道
- 核心产品/技术
- 是否属于热点方向（AI/新能源/半导体/高端制造等）

### 检索节奏与反风控要求（批量任务必守）
- 按 10-20 家为一组推进
- 单次查询间隔随机 5-20 秒
- 每组之间暂停 3-10 分钟
- 轮换百度 / Bing / 搜狗 / 官方站 / 企查类公开页面 / 招投标公示等来源
- 不要只停留在搜索结果页；尽量点进结果页阅读后再继续
- 一旦出现验证码、访问受限或明显风控，立刻切换关键词、来源或切换到 `browser-relay`

### 第五步：搜索股东和集团关系
搜索：
```
北京+[公司名称]+对外投资+子公司
北京+[公司名称]+母公司+股东
```
重点抓取：
- 投资方数量
- 对外投资数量
- 疑似控股数量
- 分支机构数量
- 是否有集团背景

### 第六步：搜索近期动态
搜索：
```
北京+[公司名称]+招聘
北京+[公司名称]+专利+融资+中标
```
重点抓取：
- 招聘动态（判断扩张/收缩）
- 专利申请（技术创新信号）
- 融资事件（是否有资本进入）
- 中标记录（大项目信号）

### 第七步：如有官网，访问官网
搜索公司官网并访问，获取：
- 主营业务详细描述
- 产品信息
- 合作伙伴
- 联系方式
- 最新动态

### 第八步：输出分析报告
按 `references/analysis-template.md` 中的模板格式输出，包含：
1. 客户基础信息
2. 核心结论摘要（3句话）
3. 企业经营与行业判断
4. 股东背景与集团关系
   - **★★★ 特别重要：母子公司辐射分析（必须填写）**
     - **向上查**：该公司是否有母公司？母公司体量/行业/融资/是否在名单里？→ 以大带小机会
     - **向下查**：该公司对外投资的子公司有哪些？子公司价值如何？→ 控股公司价值=子公司价值
     - 实控人名下是否有关联企业网络
     - 评级是否需要因辐射价值修正（参考模板辐射判断表）
5. 上下游与商业生态分析
6. 近期动态与增长信号
7. 资金流与银行业务场景分析
8. 风险与障碍判断
9. 机会分类判断
10. 营销切入建议
11. 综合评分（A/B/C/D类）——**母子公司辐射价值作为单独评分项**
12. 6行总结——**第3行专门写母子公司辐射机会**

## 评级标准

- **A类**：重点深挖，优先拜访。行业热点+扩张信号强+金融机会明确
- **B类**：持续跟踪，择机切入。有真实业务机会但时机未到或需进一步确认
- **C类**：维护观察，以结算为主。体量有限但有合作价值，以存款/结算为主
- **D类**：暂不重点投入。风险大于机会或他行深度绑定

## ⚠️ 批量任务执行规范（经验教训总结）

### 1. Cron 时间戳必须用 UTC
创建 cron 任务时，所有时间戳必须写成**纯 UTC**（不带时区后缀）。
- ❌ 错误：`"at": "2026-03-31T10:40:00+08:00"` → OpenClaw 会当作 UTC 解析，导致提前 8 小时触发
- ✅ 正确：当前 CST 10:40 = UTC 02:40 → `"at": "2026-03-31T02:40:00Z"`

### 2. Git 推送目标分支
- 默认推送到 `master`，不要默认创建 `main`
- 推送前检查 `git remote -v`，确认 remote URL 正确
- 使用 PAT 时 URL 格式：`https://github_pat_xxx@github.com/owner/repo.git`
- `git add` 前必须先清理缓存文件

### 3. Git 工作目录与 workspace 共用问题
⚠️ **重要**：`company-info` 的 git 工作目录就是 workspace 根目录。
- `git status` 的内容 = 仓库内容，**不要轻易 `git checkout --orphan`** 或重置
- 任何仓库清理前，先把 workspace 其他文件（skills/、memory/ 等）移走再操作
- 正确流程：备份仓库内容到 `/tmp/`，清空后重新初始化 git，再还原

### 4. Git 暂存前必须清理缓存文件
每次 `git add .` 前必须删除：
```
find . -name '*.sources.txt' -delete
find . -name 'progress.txt' -delete
find . -name 'README.md' -delete
```
严禁把搜索缓存、截图、临时文件带进仓库。

### 5. 推送内容原则
- 只推送 `.md` 分析报告文件 + `INDEX.md`
- 禁止进仓：`.cache/`、`browser-shots/`、`.sources.txt`、`progress.txt`

### 6. 每个批次必须独立生成 INDEX
INDEX 要求：
- 包含评级分布统计（A/B/C/D 数量）
- 包含超链接，可直接跳转到每家公司的 `.md` 文件
- 评级列可点击
- 更新后同步 push 到仓库

### 7. 评级解析注意边缘格式
部分研究员输出的评级格式不标准，解析时需处理：
- `**综合评级：**` 后面有时是同一行、有时是下一行 → 两个都要查
- "B/C之间" → 取靠近的方向归类
- ⭐星级评分体系 → 从平均星级推导：≥3.5 → B类，≥2.5 → C类
- 原始报告写"信息极度不足/无法评级" → 归入"未知"，保留但标注

### 8. 进度汇报机制
- isolated cron 的 `announce` 投递到 Telegram 有时不稳（任务状态 ok 但投递失败）
- 不要依赖 announce 作为唯一通知；每完成一批直接在主会话报告，最终完成做完整汇总

## 普惠金融注意事项

即使体量小的客户：
- 高新技术企业 ✅ → 科创贷适用
- 有稳定大客户TO-B ✅ → 供应链融资适用
- 有招投标记录 ✅ → 保函适用
- 有真实经营 ✅ → 代发/结算适用
- 有专利技术 ✅ → 轻资产评估融资适用

普惠不看规模，看：真实经营 + 稳定客户 + 技术/资产支撑
